# Korean MBTI Test Constitution

## Core Principles

### I. Mobile-First Test Experience
- 모든 기능은 모바일 사용성을 우선으로 설계해야 한다.
- 질문 읽기, 선택, 진행 상태 확인, 결과 확인이 작은 화면에서도 즉시 가능해야 한다.
- 질문 진행 중 사용자의 집중을 깨는 요소를 추가해서는 안 된다.

### II. Fixed Test Structure
- 테스트는 정확히 24개 문항으로 구성되어야 한다.
- 최종 결과는 정확히 16개 성격 유형 중 하나로 결정되어야 한다.
- 문항 수 또는 유형 수를 바꾸는 변경은 헌장 개정 수준으로 간주한다.

### III. Next.js And Pure Domain Logic
- 애플리케이션은 Next.js와 TypeScript를 사용해야 한다.
- 점수 계산, 결과 판정, 유형 매핑 로직은 UI 컴포넌트 밖에 둬야 한다.
- 스코어링 로직은 가능한 순수 함수로 작성해야 한다.
- 질문과 결과 데이터는 JSON 또는 명시적 타입 구조로 유지해야 한다.

### IV. Ad Placement Discipline
- 광고는 결과 페이지 또는 콘텐츠 페이지에서만 허용된다.
- 질문 진행 플로우에는 광고를 넣을 수 없다.
- 광고가 핵심 CTA, 결과 요약, 또는 모바일 가독성을 방해하면 안 된다.

### V. Release Quality Gates
- 완료 기준은 `lint`, `typecheck`, `tests`, `build` 통과다.
- 결과 판정과 점수 계산은 테스트 가능한 구조를 유지해야 한다.
- 재사용 가능한 UI 컴포넌트를 우선하고 중복 로직을 피해야 한다.

## Product Constraints
- 한국어 사용자 대상의 MBTI 스타일 성향 테스트 웹사이트를 구축한다.
- 서버 컴포넌트를 우선하고, 클라이언트 상태는 꼭 필요한 경우에만 사용한다.
- 결과 페이지는 검색성과 공유성을 고려한 의미 있는 메타데이터를 가져야 한다.

## Workflow And Review
- 스펙은 구현 방법보다 사용자 가치와 제품 제약을 먼저 설명해야 한다.
- 계획 문서는 스코어링 분리, 광고 위치 제약, 모바일 우선 UX를 명시적으로 검토해야 한다.
- 작업 목록에는 반드시 품질 게이트와 광고 노출 검증이 포함되어야 한다.

## Governance
- 이 헌장은 제품 요구사항, 계획, 작업 목록보다 우선한다.
- 문항 수, 결과 유형 수, 광고 배치 원칙 변경은 명시적 합의와 헌장 업데이트가 필요하다.
- 모든 기능 작업은 헌장 준수 여부를 검토한 뒤 진행한다.
