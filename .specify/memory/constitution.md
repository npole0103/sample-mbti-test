# 프로젝트 헌장

## 1. 제품 목적
- 이 프로젝트는 한국어 사용자를 위한 MBTI 스타일 성향 테스트 웹사이트를 만든다.
- 핵심 경험은 모바일 우선 UX, 빠른 로딩, 명확한 결과 설명, 부담 없는 광고 배치다.

## 2. 제품 불변 조건
- 사용자는 정확히 24개 문항에 응답해야 한다.
- 최종 결과는 정확히 16개 성격 유형 중 하나로 귀결되어야 한다.
- 광고는 결과 페이지 또는 별도 콘텐츠 페이지에만 노출할 수 있다.
- 질문 진행 중인 플로우에는 광고를 배치하지 않는다.
- 주요 CTA와 결과 요약은 모바일 첫 화면에서 쉽게 찾을 수 있어야 한다.

## 3. 아키텍처 및 코드 원칙
- 모든 구현은 TypeScript를 엄격하게 사용한다.
- 애플리케이션 프레임워크는 Next.js를 사용한다.
- 점수 계산과 유형 판정 로직은 UI 컴포넌트 바깥에 둔다.
- 점수 계산, 결과 산출, 매핑 로직은 가능한 순수 함수로 작성한다.
- 질문 데이터는 JSON 또는 명시적 타입 구조로 관리한다.
- UI는 재사용 가능한 컴포넌트로 구성하고 중복 로직을 피한다.
- 클라이언트 상태가 꼭 필요한 경우를 제외하고 서버 컴포넌트를 우선한다.

## 4. 품질 게이트
- 구현 완료 전 반드시 `lint`, `typecheck`, `tests`, `build`를 모두 통과해야 한다.
- 신규 기능은 스코어링 로직과 결과 판정 규칙을 테스트 가능한 구조로 유지해야 한다.
- 모바일 레이아웃에서 가독성과 조작성이 떨어지면 완료로 간주하지 않는다.

## 5. SEO 및 콘텐츠 원칙
- 결과 페이지는 각 유형별 의미 있는 메타데이터를 가져야 한다.
- 검색 유입과 공유를 고려한 결과 설명, 제목, 요약 구조를 유지한다.
- 광고 배치로 인해 핵심 결과 요약이 밀리거나 가려지면 안 된다.

## 6. 스펙 작성 원칙
- 스펙은 구현 방법보다 사용자 가치와 제약 조건을 먼저 설명한다.
- 모든 스펙과 계획 문서는 위 불변 조건을 위반하지 않아야 한다.
- 모호한 요구사항이 있으면 24문항, 16유형, 질문 중 광고 금지, 모바일 우선 원칙을 최우선으로 해석한다.
