# 기능 스펙: 나는 어떤 디저트형 연애일까?

## 서비스 개요

### 서비스명
- 나는 어떤 디저트형 연애일까?

### 한 줄 소개
- 사용자의 연애 성향과 관계 스타일을 24문항 테스트로 분석하여 16가지 디저트 유형 중 하나로 결과를 보여주는, 귀엽고 공유하기 쉬운 MBTI 스타일 연애 성향 테스트 웹사이트

### 핵심 컨셉
- 디저트와 빵 세계관 기반의 연애 성향 테스트
- 커플, 썸, 연애 성향 중심의 관계형 콘텐츠
- 2030 타깃의 귀엽고 감성적인 무드
- 결과 이미지 공유에 최적화된 바이럴형 테스트
- 단순 성격 테스트가 아니라 "연애할 때 나는 어떤 사람인가"를 디저트에 빗대어 표현

## 기획 의도

### 기획 배경
- 기존 MBTI형 테스트는 여전히 인기가 많지만, 단순한 성격 설명보다 연애 성향, 궁합, 데이트 스타일을 연결한 테스트가 더 높은 공유성과 몰입도를 가진다.
- 2030 세대는 카페 및 디저트 감성, 귀엽고 말랑한 캐릭터성, 결과 캡처 후 공유하기 좋은 포맷, 관계형 콘텐츠에 대한 반응이 좋다.

### 차별점
- 딱딱한 MBTI 코드 대신 디저트 캐릭터 세계관을 사용한다.
- 결과를 감성적이고 직관적으로 이해할 수 있다.
- "내 결과" 이후 "궁합 확장"까지 이어질 수 있다.
- 광고를 붙여도 상대적으로 자연스러운 가벼운 콘텐츠형 서비스다.

## 타깃 사용자

### 주요 타깃
- 20대에서 30대 초반 사용자
- 연애, 썸, 커플 콘텐츠에 관심이 많은 사용자
- MBTI, 심리 테스트, 밈형 콘텐츠 소비에 익숙한 사용자
- 인스타그램, 카카오톡, X, 스레드 등에 결과 공유를 즐기는 사용자

### 세부 타깃
- 커플 사용자: 서로 결과를 비교하고 궁합을 보고 싶어하는 사용자
- 썸 및 연애 관심층: 연애할 때 자신의 스타일이 궁금한 사용자
- 친구끼리 즐기는 테스트 소비층: 가볍게 해보고 캡처 공유하는 재미를 원하는 사용자

## 서비스 목표

### 핵심 목표
- 사용자가 가볍고 재밌게 참여할 수 있어야 한다.
- 테스트 결과가 귀엽고 공감되며 공유하고 싶어야 한다.
- 결과 화면에서 추가 체류 및 재공유가 발생해야 한다.
- 결과 페이지 중심으로 Google AdSense 수익화가 가능해야 한다.

### 성공 기준
- 테스트 이탈률이 낮아야 한다.
- 결과 페이지 공유율이 높아야 한다.
- 유형별 결과 페이지 체류시간이 확보되어야 한다.
- 광고가 UX를 해치지 않으면서 결과 페이지에 자연스럽게 녹아들어야 한다.

## 서비스 컨셉 및 톤앤매너

### 메인 세계관
- "디저트로 보는 나의 연애 성향"
- 사람마다 연애할 때의 성향은 다르며, 그 차이를 디저트의 식감, 맛, 분위기에 연결해 표현한다.
- 누군가는 겉은 바삭하지만 속은 여리고, 누군가는 담백한데 오래 갈수록 중독적이며, 누군가는 폭신하고 다정하지만 은근한 집착이 있는 식으로 결과를 설명한다.

### 비주얼 무드
- 버터 옐로우
- 크림 화이트
- 코코아 브라운
- 딸기핑크
- 베이지와 밀크톤 중심

### 감성 방향
- 너무 유아틱하지 않은 카페 감성
- "성수/연남 디저트샵 포스터" 같은 느낌
- 부드럽고 말랑한 텍스처
- 귀엽지만 살짝 세련된 톤

### 카피 톤
- 겉은 바삭, 속은 말랑
- 한입 보면 연애 성격이 보인다
- 우리 둘은 어떤 디저트 조합일까?
- 너는 어떤 디저트형 사랑일까?

## 핵심 기능 정의

### 테스트 기능
- 총 24문항을 제공해야 한다.
- 문항은 객관식 선택형이어야 한다.
- 각 문항은 2지선다 또는 4지선다 형태로 구성할 수 있다.
- 응답을 기반으로 내부 점수를 계산해 16유형 중 하나를 도출해야 한다.

### 결과 기능
- 디저트 유형명
- 성격 요약
- 연애 스타일 설명
- 장점 및 주의 포인트
- 잘 맞는 유형 및 덜 맞는 유형
- 추천 데이트 무드
- 결과 공유용 이미지 또는 텍스트

### 공유 기능
- 카카오톡 공유
- 링크 복사
- 결과 카드 이미지형 캡처에 최적화된 UI
- SNS 공유 유도 문구 제공

### 궁합 확장 기능
- 선택 기능으로 "내 유형과 잘 맞는 디저트형은?"과 "우리 둘의 디저트 궁합 보기"를 확장할 수 있어야 한다.
- 결과 페이지 하단에 관련 CTA를 배치할 수 있어야 한다.

### 수익화 기능
- Google AdSense를 삽입할 수 있어야 한다.
- 광고 위치는 결과 페이지와 정보성 페이지에만 허용된다.
- 질문 진행 화면에는 광고를 배치할 수 없다.

## 테스트 구조

### 문항 수
- 총 24문항

### 유형 수
- 총 16유형

### 성향 축 설계
- 24문항은 4개 축 x 6문항 구조를 기준으로 설계한다.
- A. 애정표현 방식: 표현의 직접성, 행동 중심 여부, 감정 전달 방식
- B. 갈등 해결 방식: 즉시 해결, 시간 두기, 감정형 또는 이성형 경향
- C. 데이트 취향: 계획형 또는 즉흥형, 활동형 또는 집콕형, 이벤트형 또는 일상형
- D. 관계 온도와 속도: 천천히 가까워짐, 빠른 친밀감, 깊고 오래감, 표현 빈도

## 결과 유형 설계 방향

### 전체 방향
- 16개 유형 모두를 디저트 및 빵 캐릭터로 구성한다.
- 최신 유행 디저트만으로 채우지 않고 오래 가는 기본 베이커리와 트렌디 디저트를 혼합한다.

### 이유
- 유행 디저트만 전면 배치하면 트렌드 수명이 짧아질 수 있다.
- 장기적으로 익숙한 디저트와 최신 감성을 반영한 트렌드 디저트를 적절히 섞는 편이 안정적이다.

### 추천 구성 비율
- 기본 베이커리 및 디저트 10개에서 12개
- 트렌드 디저트 4개에서 6개

- 버터떡형 — 무심쫀득 러버
- 쫀득쿠키형 — 은근집착 다정파
- 크루아상형 — 예민폭신 섬세파
- 소금빵형 — 담백중독 안정형
- 도넛형 — 애교만점 기분파
- 까눌레형 — 겉차분 속뜨거움형
- 스콘형 — 무던하고 믿음직한 현실파
- 타르트형 — 꾸미는 감성 로맨티스트
- 머핀형 — 포근수다 인간난로형
- 프레첼형 — 츤데레 밸런스형
- 슈크림빵형 — 속마음부자 여린형
- 식빵형 — 기본기탄탄 편안형
- 롤케이크형 — 부드러운 배려왕
- 바게트형 — 직진단호 매력형
- 휘낭시에형 — 조용한 고급취향형
- 크럼블형 — 자유분방 호기심형

## 16가지 디저트 유형 설명 초안

### 버터떡형 — 무심쫀득 러버
- 한 줄 요약: 툭툭 무심한 것 같지만 한 번 마음을 주면 오래 붙어 있는 타입
- 연애 설명: 표현은 담백한 편이지만 관계의 온도는 꾸준히 지켜낸다. 겉으로는 무던해 보여도 좋아하는 사람에게는 은근히 깊고 오래 가는 애정을 쌓는다.

### 쫀득쿠키형 — 은근집착 다정파
- 한 줄 요약: 다정함 속에 살짝의 집착이 숨어 있는 달콤쫀득한 타입
- 연애 설명: 상대를 세심하게 챙기고 작은 변화도 잘 알아차린다. 좋아질수록 더 자주 확인하고 더 가까이 있고 싶어하는 편이라 애정 표현이 꾸준하고 밀도 높다.

### 크루아상형 — 예민폭신 섬세파
- 한 줄 요약: 결은 바삭하지만 속은 누구보다 부드러운 섬세한 타입
- 연애 설명: 분위기와 말투, 감정의 미세한 결까지 잘 읽는다. 상처도 잘 받지만 그만큼 상대를 세심하게 배려하며, 서로의 감정선이 맞을 때 가장 깊게 빠진다.

### 소금빵형 — 담백중독 안정형
- 한 줄 요약: 자극적이진 않아도 오래 곁에 두고 싶은 편안한 타입
- 연애 설명: 과한 밀당보다 안정감 있는 관계를 선호한다. 처음엔 담백해 보여도 함께할수록 편안함과 신뢰가 커져 자꾸 생각나는 매력이 있다.

### 도넛형 — 애교만점 기분파
- 한 줄 요약: 사랑을 받으면 더 반짝이는 발랄하고 사랑스러운 타입
- 연애 설명: 감정 표현이 풍부하고 애교가 많아 연애 분위기를 밝게 만든다. 기분에 따라 에너지 차이가 있지만, 좋아하는 사람 앞에서는 귀엽고 솔직한 매력을 숨기지 못한다.

### 까눌레형 — 겉차분 속뜨거움형
- 한 줄 요약: 겉은 차분하고 단단하지만 속은 뜨겁게 타오르는 타입
- 연애 설명: 처음엔 쉽게 속을 보여주지 않지만 마음이 열리면 누구보다 진하게 사랑한다. 감정을 함부로 소비하지 않아 더 깊고 인상적인 관계를 만드는 편이다.

### 스콘형 — 무던하고 믿음직한 현실파
- 한 줄 요약: 화려하진 않아도 옆에 있으면 가장 든든한 타입
- 연애 설명: 말보다 행동으로 책임감과 신뢰를 보여준다. 사소한 이벤트보다 현실적인 배려와 꾸준함을 중요하게 여기며, 오래 갈 연애에서 특히 강하다.

### 타르트형 — 꾸미는 감성 로맨티스트
- 한 줄 요약: 사랑도 분위기도 예쁘게 만들고 싶은 감성 가득한 타입
- 연애 설명: 기념일, 무드, 표현의 디테일을 소중하게 생각한다. 연애를 단순한 관계가 아니라 함께 만드는 장면으로 여기는 편이라, 낭만적이고 정성 어린 순간을 잘 만든다.

### 머핀형 — 포근수다 인간난로형
- 한 줄 요약: 함께 있으면 마음이 금방 풀리는 따뜻한 타입
- 연애 설명: 대화가 자연스럽고 상대를 편하게 만들어주는 힘이 있다. 다정한 말과 리액션으로 친밀감을 빠르게 높이며, 연애에서 정서적 안정감을 크게 주는 편이다.

### 프레첼형 — 츤데레 밸런스형
- 한 줄 요약: 선은 지키지만 챙길 땐 확실히 챙기는 반전 매력 타입
- 연애 설명: 겉으로는 쿨하고 장난스럽지만 실제로는 관계의 균형을 잘 본다. 과하지 않게 다가가면서도 필요한 순간에는 누구보다 확실하게 마음을 보여준다.

### 슈크림빵형 — 속마음부자 여린형
- 한 줄 요약: 마음속 애정과 생각이 많은 여리고 다정한 타입
- 연애 설명: 겉으로 다 표현하지 못해도 속으로는 늘 상대를 많이 생각한다. 상처를 쉽게 받는 만큼 상대의 말과 태도에도 민감하지만, 그만큼 진심이 깊고 따뜻하다.

### 식빵형 — 기본기탄탄 편안형
- 한 줄 요약: 특별히 튀지 않아도 가장 자주 찾게 되는 안정형 타입
- 연애 설명: 과장 없는 배려와 일상의 편안함으로 관계를 오래 유지한다. 화려한 이벤트보다 자연스럽게 스며드는 친밀감을 중요하게 여기며, 함께할수록 가치가 커진다.

### 롤케이크형 — 부드러운 배려왕
- 한 줄 요약: 분위기를 둥글게 만들고 사람을 편하게 감싸는 타입
- 연애 설명: 갈등 상황에서도 부드럽게 말하고 상대 입장을 먼저 생각한다. 배려가 몸에 밴 사람이라 관계의 온도를 매끈하게 유지하는 데 능숙하다.

### 바게트형 — 직진단호 매력형
- 한 줄 요약: 좋아하면 헷갈리지 않게 표현하는 시원시원한 타입
- 연애 설명: 감정이 생기면 비교적 빠르고 분명하게 움직인다. 밀당이나 애매한 태도보다 솔직한 방향성을 선호하며, 연애에서도 추진력과 존재감이 강한 편이다.

### 휘낭시에형 — 조용한 고급취향형
- 한 줄 요약: 티는 많이 안 내도 기준과 감도가 분명한 타입
- 연애 설명: 사람을 쉽게 선택하지 않지만 마음을 주면 깊이 본다. 취향과 기준이 뚜렷해 관계에서도 진중함이 묻어나며, 조용하지만 오래 남는 매력을 가진다.

### 크럼블형 — 자유분방 호기심형
- 한 줄 요약: 틀에 갇히지 않고 설렘의 결을 자유롭게 즐기는 타입
- 연애 설명: 새로운 경험과 자극, 재밌는 대화를 좋아한다. 답답한 관계보다는 서로의 개성을 살려주는 연애에 끌리며, 함께 있을 때 예상 밖의 즐거움을 많이 만든다.

## UX 설계 방향

### 전체 UX 원칙
- 빠르고 가볍게 시작 가능해야 한다.
- 질문 화면은 몰입이 깨지면 안 된다.
- 결과 화면은 공유 욕구가 생겨야 한다.
- 모바일에서 가장 예쁘고 편해야 한다.

### 질문 화면 원칙
- 1문항 1화면 또는 카드 전환형 구조
- 상단 진행률 표시
- 답변 버튼은 크고 명확하게
- 여백을 충분히 확보
- 질문 중간 광고 금지
- 질문 화면 내 군더더기 UI 최소화

### 결과 화면 원칙
- 첫 화면에 유형명과 핵심 요약을 노출한다.
- 결과 카드만 캡처해도 예쁘게 보이도록 설계한다.
- CTA는 눈에 띄되 부담스럽지 않게 배치한다.
- 광고는 결과 해석 이후 자연스럽게 노출한다.

## 광고 및 수익화 정책

### 광고 삽입 원칙
- 질문 흐름 중 광고 금지
- 결과 페이지에만 주요 광고 삽입
- 정보성 상세 페이지에도 광고 삽입 가능
- 사용자 경험을 해칠 정도의 과도한 광고 배치 금지

### 추천 광고 위치
- 결과 요약 아래 1개
- 상세 설명 하단 1개
- 추가 정보 또는 유형 궁합 섹션 중간이나 하단 1개

### 광고 UX 원칙
- CTA 버튼과 겹치지 않아야 한다.
- 결과 핵심 영역을 밀어내면 안 된다.
- 모바일에서 과한 스크롤을 유발하면 안 된다.

## 확장 콘텐츠 전략

### 확장 가능 기능
- 우리 둘의 디저트 궁합 테스트
- 유형별 데이트 추천
- 유형별 연락 스타일 분석
- 유형별 연애 밸런스 게임
- 친구, 썸, 연인 버전 파생 테스트

### 정보성 콘텐츠 예시
- 버터떡형과 잘 맞는 디저트는?
- 쫀득쿠키형 연애 특징 총정리
- 디저트 유형별 싸울 때 반응
- 디저트 유형별 데이트 취향

### 확장 이유
- SEO에 유리하다.
- 광고 삽입 지면을 늘려준다.
- 결과 페이지 이후 체류 시간을 늘려준다.

## 기술 구현 관점

### 기본 기술 방향
- 웹 기반 반응형 테스트 사이트
- 모바일 우선 설계
- 결과는 데이터 기반으로 계산
- 질문 및 결과 데이터는 관리 가능한 구조로 분리

### 데이터 구조 방향
- 질문 데이터 JSON 분리
- 선택지별 점수 매핑 구조 분리
- 결과 유형 프로필 데이터 분리
- 결과 카드 및 공유 데이터 분리 가능

### 비즈니스 로직 원칙
- 점수 계산은 UI에서 분리해야 한다.
- 순수 함수 형태로 계산 가능해야 한다.
- 16유형 결과는 결정론적으로 도출되어야 한다.
- 테스트와 유지보수가 쉬운 구조로 구현해야 한다.

## 브랜딩 및 바이럴 전략

### 바이럴 포인트
- 결과명이 귀엽고 기억에 남아야 한다.
- 공유 이미지가 예뻐야 한다.
- "헐 너 완전 소금빵형이야" 같은 대화가 나와야 한다.
- 친구나 연인에게 링크를 보내기 쉬워야 한다.

### 추천 메시지 포맷
- 나 연애할 때 버터떡형이래
- 우리 둘 디저트 궁합 뭐 나오는지 해봐
- 겉바속촉인 줄 알았는데 완전 쫀득쿠키형이네

### 서비스 네이밍 방향
- 메인 서비스명은 "나는 어떤 디저트형 연애일까?"로 유지한다.
- 공유용 서브 카피는 여러 버전으로 실험할 수 있다.
